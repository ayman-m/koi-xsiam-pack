#### Playbooks

##### Koi Unified - Execute Endpoint Script

Fixed a bug where every run silently targeted only 30 endpoints. The endpoint query passed no `limit`,
so `core-get-endpoints` applied its default page size of 30 regardless of how many endpoints were in
the target group. On a large group each run reached a rotating 30 and the rest were never scanned.
- Added a **max_endpoints** input, passed as the query's `limit`, overridable per List entry with
  `target.max_endpoints`.
- **Verified cap: `core-get-endpoints` rejects `limit` above 100** ("Search size must fulfill the
  requirement: 0 < search_size <= 100", HTTP 500). The default is therefore **100** — the safe maximum
  for one run, and over 3x the previous silent cap of 30. **Do not raise it above 100.**
- Covering a group larger than 100 in a single run requires paginating the endpoint query with
  `page`/`page_size` and, depending on the per-action endpoint cap of `core-script-run`, batching the
  execution. That is a follow-up change that should be built and tested against the target tenant's
  Cortex Core - IR version.

Fixed false-positive failure notifications. `core-script-run` dispatches an action and then polls it;
when polling expired the playbook treated that as an execution failure even though the action was still
running in Action Center and usually completed.
- Added a **dispatch check** on the error path: if an `action_id` exists, the run is reported as
  `timed_out` rather than failed.
- Added `ScriptResult.timed_out` and `ScriptResult.endpoint_count` outputs.

##### Koi Unified - Process Config Entry

- A timed-out run is now recorded as an informational **STILL RUNNING** entry instead of sending a
  failure email. Only a genuine dispatch failure — where no action was created — notifies.
- Notification emails now include the endpoint count and the Action ID. The previous failure mail gave
  only a reason string, with no way to find the corresponding action.
- The failure subject is now **FAILED TO DISPATCH**, distinguishing it from a run still in progress.
- Passes `target.max_endpoints` through to the execution sub-playbook.
