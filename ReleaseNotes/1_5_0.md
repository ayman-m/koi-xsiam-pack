#### Scripts

##### New: KoiScanTracker

Endpoint scan-coverage ledger that lets the Script Runner cover endpoint groups larger than the
100-endpoint `core-get-endpoints` cap. Maintains a compact CSV List (endpoint_id,last_scan-epoch)
and is action-dispatched: **refresh** (enumerate the group via the Core REST API `core-api-post`,
paged past 100, and upsert membership), **select** (return up to `max` endpoints that are due —
never scanned, or `last_scan` older than `rescan_interval_hours` — and currently connected), and
**mark** (stamp `last_scan=now`). Requires the **Core REST API** integration to be configured.
The **select** connected-check also filters by `endpoint_os`, so a wrong-OS row that somehow lands
in a tracker can never be dispatched to.

#### Playbooks

##### New: Koi Unified - Refresh Tracker / Koi Unified - Refresh Entry

Refresh job that walks every configuration entry's endpoint group and upserts membership into that
entry's tracker List. Attach to a time-triggered Job that runs less often than the scan job.

##### Koi Unified - Execute Endpoint Script

Reworked to a tracker-driven scan: **select** the due, connected endpoints from the tracker, run the
script on them, then **mark** them scanned — mark-on-dispatch, so it no longer waits on Action
Center completion (removing the false-positive-failure path by construction). New inputs
`tracker_list`, `rescan_interval_hours`, and `endpoint_os` (passed to select for the OS-safe
connected-check); dropped the group/hostname enumeration inputs.

##### Koi Unified - Process Config Entry

Passes `target.tracker_list`, `target.rescan_interval_hours`, and `target.endpoint_os` to the executor.

##### Configuration List

New per-entry fields: **`target.tracker_list`** (the CSV List this scope's coverage is tracked in,
e.g. "Koi Scan Tracker - Windows") and **`target.rescan_interval_hours`** (hours before a scanned
endpoint is due again; default 720). The existing **`target.endpoint_os`** now also drives the
OS-safe select, so each scope only ever scans endpoints of its own platform.
